# 迭代123 - 优化行情展示功能需求

## 1. 需求背景

当前系统缺少一个统一的“行情报价”页面，用户无法在一个页面内快速查看不同交易接口下的实时行情数据，也无法像专业交易软件一样对行情列表进行搜索、筛选、排序和快速查看图表。

结合原型图，本次迭代的核心目标不是做一个“全市场扫描系统”，而是先补齐一个**专业行情终端风格的统一行情页**，满足以下高频场景：

- **多接口切换查看行情**
- **常用品种快速浏览**
- **自定义维护关注品种**
- **按涨跌幅、成交量等维度快速筛选与排序**
- **从行情列表进入单品种图表详情**

## 2. 当前架构对齐与设计约束

为避免需求与当前系统实现脱节，本次需求需与现有架构保持一致：

- **页面能力应优先复用现有网关状态体系**
  - 项目中已存在网关状态、网关连接、已连接网关查询相关能力。
  - “行情报价”页面应复用现有网关可用性状态，而不是独立维护一套连接状态。

- **数据源显示名与后端标识需要解耦**
  - 前端可展示 `CTP`、`IB`、`MT5`、`Binance`、`OKX`。
  - 后端实际标识需允许映射，例如 `IB` 在当前系统中更接近 `IB_WEB`。

- **不同数据源能力并不完全等价**
  - 不同接口在可用字段、刷新方式、图表能力、品种搜索能力上可能不同。
  - 页面必须基于“能力可用性”做降级，而不能假设所有数据源字段完全一致。

- **Binance / OKX 需要显式定义能力边界**
  - 当前架构下，这两个数据源的网关接入能力与完整行情 runtime 能力不一定已经齐备。
  - 若本次迭代无法提供稳定行情数据，则前端需显示“未接入 / 接入中 / 不可用”状态，而不是伪装为可用。

- **实时更新采用“推送优先、轮询兜底”**
  - 若后端已支持推送，则优先使用推送。
  - 若未支持推送，则采用轮询。
  - 无论采用哪种方式，前端都必须展示当前刷新状态与最近更新时间。

## 3. 本次迭代范围

### 3.1 P0 范围

本次迭代必须完成以下闭环：

- 左侧新增 `行情报价` 入口
- 按数据源切换行情列表
- 展示默认品种与自定义品种的统一行情表格
- 支持当前列表内的搜索、筛选、排序
- 支持手动刷新与自动刷新
- 支持自定义添加 / 删除品种
- 支持异常状态、空状态、不可用状态
- 支持关键配置持久化

### 3.2 P1 范围

以下能力作为增强项，允许在 P0 完成后交付：

- 图表详情区域
- 列显隐与列顺序配置
- 更丰富的筛选条件
- 更细粒度的刷新状态与视觉反馈

### 3.3 本次不做

以下内容不属于本次迭代目标：

- 深度行情、盘口十档、逐笔成交
- 全市场海量扫描与跨交易所统一排名
- 自定义技术指标体系
- 多自选分组体系
- 复杂预警、提醒、订阅中心
- 图表高级绘图工具

## 4. 需求目标

- **统一入口**
  - 用户可在左侧导航进入统一的行情报价页面。

- **统一切换**
  - 用户可在同一页面切换不同数据源查看行情。

- **统一列表**
  - 用户可在高信息密度表格中查看行情，并获得一致的字段展示体验。

- **统一操作**
  - 用户可对当前列表执行搜索、筛选、排序、刷新、自定义品种管理。

- **统一状态反馈**
  - 用户可清楚感知当前数据源是否可用、行情是否新鲜、刷新是否失败。

## 5. 用户角色与核心场景

### 5.1 用户角色

- **交易员 / 操盘用户**
  - 快速查看多个交易接口下的关注品种报价。

- **策略运行观察者**
  - 结合网关运行状态观察关键品种行情是否正常更新。

### 5.2 核心场景

- **场景A：切换数据源**
  - 用户希望在 `CTP`、`IB`、`MT5`、`Binance`、`OKX` 之间快速切换，查看各自关注品种。

- **场景B：维护自选品种**
  - 用户希望针对不同数据源维护不同的关注品种列表，并在下次进入时恢复。

- **场景C：快速定位行情**
  - 用户希望对当前列表按代码、名称、涨跌幅、成交量等条件快速筛选或排序。

- **场景D：查看单品种走势**
  - 用户希望从表格进入图表详情，快速查看该品种的 K 线、分时与成交量。

## 6. 页面入口与布局

### 6.1 页面入口

- **导航位置**
  - 左侧导航“首页”下方新增 `行情报价` 菜单。

- **点击行为**
  - 点击后进入行情报价页面。

### 6.2 页面整体布局

页面整体采用与原型图一致的**深色高密度表格风格**，建议分为以下区域：

#### 6.2.1 顶部数据源切换区

- 显示数据源标签：
  - `CTP`
  - `IB`
  - `MT5`
  - `Binance`
  - `OKX`
- 标签应支持以下状态：
  - 可用
  - 未配置
  - 未连接
  - 接入中 / 暂不可用
- 默认进入页面时：
  - 优先展示用户上一次选择的数据源
  - 若无历史记录，则展示第一个可用数据源
  - 若无任何可用数据源，则展示统一空状态与引导

#### 6.2.2 工具栏区域

工具栏包含以下能力：

- **搜索框**
  - 用于在“当前已展示列表”内按代码 / 名称模糊搜索

- **筛选入口**
  - 用于在“当前已展示列表”内按分类、涨跌状态等条件筛选

- **排序能力**
  - 优先通过表头排序实现
  - 也可保留独立排序入口

- **刷新按钮**
  - 手动刷新当前数据源下的行情列表

- **自动刷新开关**
  - 开启 / 关闭自动刷新

- **自动刷新频率**
  - 例如：`1秒 / 3秒 / 5秒 / 10秒 / 30秒`

- **添加品种按钮**
  - 打开添加品种弹窗或侧栏

- **列设置按钮**
  - 作为 `P1` 能力提供

#### 6.2.3 行情表格区域

- 以高信息密度表格展示当前数据源的行情
- 支持纵向滚动
- 支持表头排序
- 支持固定关键列，例如代码、名称
- 支持单行选中态
- 涨跌字段使用颜色区分：
  - 上涨：红色
  - 下跌：绿色
  - 平盘：中性色

#### 6.2.4 图表详情区域

- 作为 `P1` 能力提供
- 进入方式：
  - 点击表格行
  - 或点击“查看图表”操作
- 展现形式：
  - 优先采用抽屉或页面下半区展开
- 支持：
  - `K线图`
  - `分时图`
  - `成交量图`

## 7. 功能需求

### 7.1 数据源切换

#### 7.1.1 数据源范围

前端显示层面支持以下标签：

- `CTP`
- `IB`
- `MT5`
- `Binance`
- `OKX`

后端需支持“显示名称”与“实际标识”映射，例如：

- `IB` -> `IB_WEB`
- `Binance` -> `BINANCE`

#### 7.1.2 切换规则

- 点击数据源标签后，重新加载对应数据源的行情列表
- 每个数据源维护各自独立的：
  - 默认品种
  - 自定义品种
- 每个数据源维护各自独立的界面配置：
  - 最近一次排序方式
  - 自动刷新开关与频率
  - 列配置（`P1`）
- 切换数据源时，不应串用上一个数据源的列表数据与异常状态

#### 7.1.3 数据源状态定义

数据源标签至少支持以下状态：

- **可用**
  - 网关已连接，且具备行情查询能力

- **未配置**
  - 用户尚未完成该数据源必要配置

- **未连接**
  - 已配置但当前网关未连接

- **接入中 / 暂不可用**
  - 已在系统中占位，但当前版本不保证具备稳定行情能力

不同状态下的交互要求：

- `可用`
  - 可进入并加载行情列表

- `未配置`
  - 点击后显示配置引导

- `未连接`
  - 点击后显示连接提示与重试入口

- `接入中 / 暂不可用`
  - 可展示说明文案，但不进入伪造行情页

#### 7.1.4 默认展示逻辑

- 默认优先展示用户上一次选中的数据源
- 若上一次数据源当前不可用，则自动切换到第一个可用数据源
- 若没有任何可用数据源，则展示统一空状态：
  - 当前无可用行情数据源
  - 引导用户前往网关连接或配置页面

### 7.2 自定义品种管理

#### 7.2.1 目标定义

- “自定义品种”本质上是当前数据源下的**关注品种列表**
- `P0` 不做多分组自选，仅维护单一关注列表
- 当前数据源的默认品种与自定义品种合并后形成“当前展示列表”

#### 7.2.2 添加品种

- 用户可在当前数据源下添加其他品种
- 添加方式支持：
  - 输入代码精确添加
  - 通过代码 / 名称搜索后选择添加
- 添加成功后：
  - 立即进入当前展示列表
  - 自动触发一次该品种行情拉取
  - 在列表中高亮或滚动定位为增强项，可后续补充

#### 7.2.3 校验规则

- 不允许重复添加同一品种
- 若代码不存在，需提示“品种不存在”
- 若品种存在但不属于当前数据源，需提示“当前数据源不支持该品种”
- 若当前数据源无品种搜索能力，可退化为仅支持代码精确添加

#### 7.2.4 删除与持久化

- 支持从当前列表中移除已添加的自定义品种
- 默认品种不可被物理删除；如需隐藏，属于后续增强能力
- 自定义品种列表需按“用户 + 数据源”维度持久化
- 页面刷新或重新进入后，应恢复上次自定义品种列表

### 7.3 行情表格展示

#### 7.3.1 展示原则

- 行情页以高信息密度表格为核心
- `P0` 的搜索、筛选、排序都仅作用于**当前已加载的展示列表**
- `P0` 不提供全市场分页检索，也不提供跨交易所统一榜单

#### 7.3.2 默认字段

默认建议展示以下字段：

- **序号**
- **代码**
- **名称**
- **分类 / 交易所**
- **最新价**
- **涨跌**
- **涨跌幅**
- **买价**
- **卖价**
- **最高价**
- **最低价**
- **开盘价**
- **昨收 / 前收**
- **成交量**
- **成交额**
- **持仓量**（如适用）
- **更新时间**

#### 7.3.3 字段兼容性

- 前后端需定义统一字段口径
- 不同接口缺失字段允许为空
- 前端对缺失字段统一展示为 `--`
- 接口特有字段不进入 `P0` 默认列，可在后续扩展列中按需开放

#### 7.3.4 表格交互能力

- 支持表头排序
- 支持固定关键列，例如代码、名称
- 支持纵向滚动
- 支持单行选中态
- 行点击可作为 `P1` 图表详情入口
- 行情数值更新时，建议提供轻量视觉反馈，但不作为 `P0` 必须项

#### 7.3.5 涨跌颜色规则

- 使用与原型风格一致的“红涨绿跌”约定：
  - 上涨：红色
  - 下跌：绿色
  - 平盘：中性色
- 对于无昨收或无参考价的数据，不强行计算涨跌颜色

### 7.4 搜索、筛选、排序

#### 7.4.1 搜索

- 支持按以下维度进行模糊搜索：
  - 代码
  - 名称
- 搜索范围仅限当前已展示列表
- 输入后应实时刷新筛选结果

#### 7.4.2 筛选

`P0` 建议支持以下筛选条件：

- 分类
- 上涨 / 下跌 / 平盘
- 自定义品种 / 全部

以下筛选作为 `P1` 增强项：

- 涨跌幅区间
- 成交量区间
- 是否有持仓量字段

#### 7.4.3 排序

支持按以下字段排序：

- 最新价
- 涨跌
- 涨跌幅
- 成交量
- 成交额
- 持仓量
- 更新时间

#### 7.4.4 组合使用

- 搜索、筛选、排序支持组合使用
- 刷新后应尽量保留当前的搜索、筛选、排序条件
- 切换数据源后，应恢复该数据源上一次使用的筛选与排序状态

### 7.5 刷新机制

#### 7.5.1 手动刷新

- 页面提供手动刷新按钮
- 点击后刷新当前数据源下的行情数据
- 刷新完成后显示最近刷新时间

#### 7.5.2 自动刷新

- 支持自动刷新开关
- 支持设置自动刷新间隔
- 建议频率：
  - `1秒`
  - `3秒`
  - `5秒`
  - `10秒`
  - `30秒`

#### 7.5.3 刷新实现规则

- 若后端支持推送，则优先使用推送
- 若后端不支持推送，则采用轮询
- 页面在任一时刻仅允许存在一个活跃刷新任务
- 上一次刷新未完成时，不应无上限堆积新的刷新请求
- 页面离开、数据源切换、组件卸载时，必须正确清理刷新任务

#### 7.5.4 刷新状态反馈

页面需明确展示以下信息：

- 当前刷新方式：
  - 实时推送
  - 自动轮询
  - 手动刷新
- 最近更新时间
- 当前请求状态：
  - 加载中
  - 成功
  - 失败
- 若更新时间明显滞后，应提示数据可能已过期

### 7.6 图表展示

#### 7.6.1 范围定位

- 图表详情作为 `P1` 能力
- `P0` 只要求在结构上预留图表入口，不要求完整交付

#### 7.6.2 进入方式

- 用户点击行情表中的某一行或“查看图表”操作后，进入图表详情

#### 7.6.3 图表类型

支持以下图表：

- **K线图**
- **分时图**
- **成交量图**

#### 7.6.4 图表能力

- 图表需与当前选中的数据源、品种保持一致
- `K线图` 建议支持周期切换，例如：
  - `1m`
  - `5m`
  - `15m`
  - `30m`
  - `1h`
  - `4h`
  - `1d`
- 若当前数据源不支持某类图表，应明确展示“不支持”，而不是空白

#### 7.6.5 图表状态

- 加载中：显示骨架屏或加载态
- 无数据：显示空状态
- 加载失败：显示错误提示与重试入口

### 7.7 状态与异常处理

需要覆盖以下场景：

- 当前数据源未配置
- 当前数据源未连接
- 当前数据源暂不支持行情
- 当前数据源暂无行情数据
- 用户添加了无效品种
- 行情刷新失败
- 图表数据获取失败
- 某些字段缺失

对应要求：

- 页面不能白屏
- 需要有明确文案
- 需要支持用户重试
- 错误态与空状态需和“未配置 / 未连接 / 不可用”严格区分

### 7.8 用户配置持久化

以下配置建议持久化保存：

- 最近一次选择的数据源
- 每个数据源下的自定义品种列表
- 每个数据源的排序方式
- 自动刷新开关与刷新频率
- 列显隐与列顺序（`P1`）

持久化策略要求：

- 优先按用户维度保存
- 若当前阶段暂无用户级服务端存储，可临时使用前端本地存储兜底
- 一旦使用本地存储，需明确字段命名与版本策略，避免后续升级冲突

## 8. 数据字段统一建议

### 8.1 统一字段

为了兼容不同接口，建议前后端统一以下基础字段：

- `source`
- `source_label`
- `symbol`
- `name`
- `exchange`
- `category`
- `last_price`
- `change`
- `change_pct`
- `bid_price`
- `ask_price`
- `high_price`
- `low_price`
- `open_price`
- `prev_close`
- `volume`
- `turnover`
- `open_interest`
- `update_time`
- `status`
- `error_message`

### 8.2 字段口径说明

- `source`
  - 后端内部数据源标识，例如 `CTP`、`IB_WEB`、`MT5`

- `source_label`
  - 前端展示名称，例如 `IB`

- `symbol`
  - 当前数据源下的标准品种代码，作为列表主键之一

- `change`
  - 价格涨跌额；若缺少可计算基准，则允许为空

- `change_pct`
  - 涨跌幅；若缺少 `prev_close` 等基准字段，则允许为空

- `update_time`
  - 行情记录最近更新时间，需统一时区与格式约定

- `status`
  - 当前行行情状态，例如正常、缺失、错误

### 8.3 字段兼容原则

- 非所有接口都必须返回全部字段
- 缺失字段允许为空
- 前端统一做兼容展示
- 字段是否可排序、可筛选，需要由前端结合字段可用性决定

## 9. 交互流程

### 9.1 页面初始化

- 用户进入“行情报价”页面
- 系统读取上次使用的数据源与配置
- 选择第一个可用数据源或恢复上次使用的数据源
- 加载当前数据源下的默认品种和自定义品种
- 拉取行情数据并展示
- 若开启自动刷新，则自动启动刷新任务

### 9.2 切换数据源

- 用户点击新的数据源标签
- 页面切换到对应数据源
- 加载该数据源下的品种列表和行情数据
- 恢复该数据源上一次的个性化配置
- 清理上一个数据源残留的刷新任务与错误态

### 9.3 添加品种

- 用户点击“添加品种”
- 输入代码或名称搜索
- 选择目标品种后加入列表
- 列表立即刷新并展示新增品种
- 若失败，则保留弹窗状态并给出具体原因

### 9.4 刷新行情

- 用户点击手动刷新或等待自动刷新触发
- 系统拉取当前展示列表对应的最新行情
- 页面更新最近更新时间与状态提示
- 若刷新失败，则保留旧数据，并显示失败提示

### 9.5 查看图表

- 用户点击行情列表中的某个品种
- 页面打开该品种的图表详情
- 用户可在图表中切换 `K线`、`分时`、`成交量` 等视图

## 10. 非功能要求

### 10.1 性能要求

- 当前展示列表在常规关注品种规模下应保持流畅滚动与稳定刷新
- 自动刷新不应造成明显卡顿、闪烁或请求堆积
- 切换数据源时不应出现长时间白屏

### 10.2 可用性要求

- 页面必须提供明确的加载态、空态、异常态
- 用户应能从异常态快速恢复，例如重试刷新、切换数据源、前往网关连接页

### 10.3 可扩展性要求

- 数据源接入应基于统一字段与能力映射
- 后续新增数据源时，不应要求重写整个行情页
- 图表能力、列配置能力、自选分组能力应可在后续迭代增量扩展

## 11. 验收标准

### 11.1 P0 验收

- **入口可见**
  - 左侧导航存在“行情报价”菜单，且可正常进入页面

- **数据源状态可识别**
  - 用户可区分 `可用`、`未配置`、`未连接`、`暂不可用` 四类状态

- **数据源可切换**
  - 可在已接入的数据源之间切换，且切换后正确加载对应列表

- **默认品种可展示**
  - 进入可用数据源后，可看到默认品种或恢复后的关注列表

- **自定义品种可维护**
  - 支持新增、删除并持久化保存

- **搜索可用**
  - 可按代码、名称在当前展示列表内检索目标品种

- **基础筛选可用**
  - 可按分类、涨跌状态等条件筛选当前展示列表

- **排序可用**
  - 可按价格、涨跌幅、成交量等字段排序

- **刷新可用**
  - 支持手动刷新与自动刷新，且能看到最近刷新时间和刷新状态

- **异常可处理**
  - 接口未连接、无数据、加载失败、字段缺失等场景下有明确提示，页面不白屏

### 11.2 P1 验收

- **图表可用**
  - 支持查看 `K线图`、`分时图`、`成交量图`

- **列配置可用**
  - 支持列显隐、列顺序调整及持久化恢复

- **增强筛选可用**
  - 支持涨跌幅区间、成交量区间等高级筛选

## 12. 风险与待确认项

以下事项需在开发前确认，否则容易影响排期：

- **Binance / OKX 行情能力边界**
  - 当前版本是否具备稳定行情 runtime
  - 若不具备，页面仅展示状态还是接入第三方行情接口

- **IB 命名映射**
  - 前端是否统一展示为 `IB`
  - 后端是否统一使用 `IB_WEB`

- **品种搜索来源**
  - 是来自各网关自身能力，还是平台维护的品种静态清单

- **持久化落点**
  - 使用服务端用户配置存储，还是前端本地存储临时兜底

- **图表复用方案**
  - 是否复用现有 `Kline` 相关组件
  - 是否需要统一图表接口协议

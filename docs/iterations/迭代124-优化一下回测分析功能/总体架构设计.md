# 迭代124 - 回测分析功能优化 - 总体架构设计

> **文档状态**: 已对齐优化版实施计划  
> **最后更新**: 2026-03-15

## 1. 设计目标

参考 TBQuant 的策略研究功能，在 **不破坏现有回测/优化架构** 的前提下，新增"工作区驱动"的策略研究能力。**不是一次性复制 TBQuant 全部功能**，而是分阶段交付可闭环的核心链路。

核心改动：

1. **工作区（Workspace）管理** — 用户可创建多个独立工作区，每个工作区包含多个策略单元
2. **策略单元（Strategy Unit）管理** — 每个策略单元绑定一个策略+品种+周期+参数组合，可批量运行
3. **参数优化接入** — 在工作区内对策略单元发起优化（复用现有优化引擎），结果以表格/图表展示
4. **多任务组合报告** — [P1/可顺延] 聚合多个策略单元的回测结果，生成组合绩效报告

### 2.2 复用边界（关键）

工作区 **不重写** 回测/优化执行引擎，而是：

- `WorkspaceService.run_units()` → 将策略单元映射为 `BacktestRequest` → 调用 `BacktestService.run_backtest()`
- `WorkspaceService.run_optimization()` → 调用现有 `submit_optimization()` / `param_optimization_service`
- 回测/优化结果通过 `task_id` 引用现有 `BacktestResultModel` / `OptimizationTask`，不做全量复制

## 4. 核心概念映射（TBQuant → backtrader_web）

## 5. 技术栈

- **前端**: Vue 3 + TypeScript + Element Plus + TailwindCSS + ECharts + Pinia
- **后端**: FastAPI + SQLAlchemy + SQLite/PostgreSQL
- **回测引擎**: Backtrader (复用现有 `BacktestService`)
- **优化引擎**: 复用现有 `param_optimization_service`
- **分析库**: fincore (复用现有 FincoreAdapter)
- **通信**: REST API + 轮询（基础）+ WebSocket（增强，非阻塞首版交付）

## 6. 设计原则

1. **向下兼容** — 现有 BacktestPage 和 OptimizationPage 完全保留，工作区是新增入口
2. **复用优先** — 运行回测复用 `BacktestService.run_backtest()`，优化复用 `submit_optimization()`，不重写
3. **引用而非复制** — 结果数据通过 `task_id` 引用现有表，只存关键指标快照，不做全量复制
4. **先闭环再丰富** — P0 交付主链路闭环（工作区→单元→运行→状态），P1 补充优化图表和报告，P2 顺延高级功能
5. **状态聚合** — `Workspace.status` 不独立维护，由子单元 `run_status` 聚合计算
6. **降级策略** — WebSocket 优先轮询可用，增强版后补；大表格先保证基本可用，虚拟滚动后优化

## 7. 本迭代不做项（明确排除）

- 重写现有回测执行引擎
- 重写现有优化执行引擎
- XML 导入导出
- 右键菜单、列拖拽、复杂快捷键等桌面化交互
- 完整 TBQuant 指标全集一次性对齐
- 样本外递进检验
- 自定义字段计算公式
- 为截图一致性引入复杂桌面端交互能力

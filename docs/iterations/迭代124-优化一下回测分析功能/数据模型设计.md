> **文档状态**: 已对齐优化版实施计划

> **核心变更**: 结果表由全量复制改为引用+快照模式，Workspace.status 改为聚合计算

## 1. 新增数据库表

### 1.1 workspaces（工作区表）

```sql
CREATE TABLE workspaces (
    id              VARCHAR(36)  PRIMARY KEY,  -- UUID
    user_id         VARCHAR(100) NOT NULL,     -- 所属用户
    name            VARCHAR(200) NOT NULL,     -- 工作区名称
    description     TEXT,                      -- 工作区描述
    -- 注意: status 不再作为独立字段存储，改为从子单元聚合计算（见下方说明）
    settings        JSON,                      -- 工作区全局设置(并行线程等)
    created_at      DATETIME     NOT NULL,
    updated_at      DATETIME     NOT NULL,

    INDEX idx_workspaces_user (user_id)
);
```

> **状态聚合规则** (不存字段，由查询时计算):
> - 任一单元 `running/queued` → 工作区 `running`
> - 全部 `completed` → 工作区 `completed`
> - 存在 `failed` 且无运行中 → 工作区 `error`
> - 其他 → `idle`

**settings JSON 结构:**
```json

  "serial_threads": 32,

  "parallel_task_threads": 1,
  "default_initial_cash": 1000000,
  "default_commission_rate": 0.0002,
  "report_calculation_method": "geometric"
}
```

### 1.2 strategy_units（策略单元表）

CREATE TABLE strategy_units (

    group_name      VARCHAR(100) NOT NULL,     -- 组名（分组标签）

### 1.3 unit_backtest_results（单元回测结果引用表） — 引用+快照模式

> **设计修订说明**: 不再存储完整回测结果副本（权益曲线、交易明细等），  
> 只存 `task_id` 引用和关键指标快照。详细数据从现有 `BacktestResultModel` 读取。

```sql
CREATE TABLE unit_backtest_results (
    id              VARCHAR(36)  PRIMARY KEY,
    unit_id         VARCHAR(36)  NOT NULL,      -- 所属策略单元
    workspace_id    VARCHAR(36)  NOT NULL,       -- 冗余：所属工作区
    task_id         VARCHAR(36)  NOT NULL,       -- 关联的 backtest_tasks.id（核心引用）
    run_number      INT          DEFAULT 1,      -- 第几次运行
    
    -- 关键指标快照（用于列表展示，避免每次查现有表）
    metrics_snapshot JSON,                       -- 见下方 JSON 结构
    
    status          VARCHAR(20) DEFAULT 'pending',  -- pending/completed/failed
    error_message   TEXT,
    created_at      DATETIME    NOT NULL,

    FOREIGN KEY (unit_id) REFERENCES strategy_units(id) ON DELETE CASCADE,
    FOREIGN KEY (workspace_id) REFERENCES workspaces(id) ON DELETE CASCADE,
    INDEX idx_results_unit (unit_id),
    INDEX idx_results_workspace (workspace_id)
);
```

### 1.4 unit_optimization_results（单元优化结果引用表） — 引用+摘要模式

> **设计修订说明**: 不自建复杂结果存储，只记录引用关系和摘要。  
> 完整优化结果从现有 `OptimizationTask` 读取或做轻量缓存。

```sql
CREATE TABLE unit_optimization_results (
    id              VARCHAR(36)  PRIMARY KEY,
    unit_id         VARCHAR(36)  NOT NULL,
    workspace_id    VARCHAR(36)  NOT NULL,
    task_id         VARCHAR(36)  NOT NULL,       -- 关联的 optimization_tasks.id（核心引用）
    
    -- 配置快照
    objective       VARCHAR(50),                 -- 优化目标
    config_snapshot JSON,                        -- 优化配置快照（参数范围等）
    
    -- 结果摘要
    total_combinations INT,
    completed       INT,
    failed          INT,
    status          VARCHAR(20) DEFAULT 'pending',
    best_params     JSON,                        -- 最佳参数快照
    best_metrics    JSON,                        -- 最佳参数对应的关键指标
    
    created_at      DATETIME    NOT NULL,
    updated_at      DATETIME    NOT NULL,

    FOREIGN KEY (unit_id) REFERENCES strategy_units(id) ON DELETE CASCADE,
    INDEX idx_opt_results_unit (unit_id)
);
```

> **读取策略**: 列表展示用 `best_params` + `best_metrics`；完整优化结果列表通过 `task_id` 从现有优化任务结果读取。

### 1.5 workspace_reports（工作区组合报告表）

```sql
CREATE TABLE workspace_reports (
    id              VARCHAR(36)  PRIMARY KEY,
    workspace_id    VARCHAR(36)  NOT NULL,
    name            VARCHAR(200),                -- 报告名称
    
    -- 报告配置
    stat_start_date VARCHAR(30),
    stat_end_date   VARCHAR(30),
    max_invest_cash FLOAT DEFAULT 600,           -- 组合最大投入资金(万)
    calculation_method VARCHAR(20) DEFAULT 'arithmetic',
    report_weight   VARCHAR(20) DEFAULT 'equal', -- 权重方式
    custom_fields   JSON,
    
    -- 报告单元列表
    report_units    JSON,                        -- [{unit_id, source, data_source, ...}]
    
    -- 聚合结果
    summary_metrics JSON,                        -- 汇总行指标
    
    status          VARCHAR(20) DEFAULT 'pending',
    created_at      DATETIME    NOT NULL,
    updated_at      DATETIME    NOT NULL,

    FOREIGN KEY (workspace_id) REFERENCES workspaces(id) ON DELETE CASCADE,
    INDEX idx_reports_workspace (workspace_id)
);
```

## 5. 与现有数据的关系

- `strategy_units.strategy_id` → 引用 `strategies/` 目录下的策略模板，与 `backtest_tasks.strategy_id` 相同
- `unit_backtest_results.task_id` → 可选关联 `backtest_tasks.id`，复用现有回测执行流程
- `unit_optimization_results.task_id` → 可选关联 `optimization_tasks.id`
- 工作区数据完全独立于现有表，不破坏向下兼容

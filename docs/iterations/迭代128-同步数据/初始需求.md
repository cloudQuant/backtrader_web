# 迭代128 - 本地与远程 MySQL 数据同步

> **文档状态**: 需求分析完成（V2）  
> **最后更新**: 2026-04-13  
> **迭代目标**: 在数据管理中新增"数据同步"Tab，实现本地 MySQL 与远程 Docker MySQL 之间的双向数据库同步

---

## 1. 背景

项目已通过 Docker 在远程 Ubuntu 服务器上部署成功（`docker-compose.prod.yml`），MySQL 8 运行在 `backtrader_mysql` 容器中。本地开发环境使用 Homebrew 安装的 MySQL。

当前痛点：
- 本地开发/调试产生的数据无法便捷地同步到服务器
- 服务器上的生产数据无法便捷地拉取到本地
- 手动操作 mysqldump + scp + docker exec 链路繁琐且易出错

### 同步范围

本迭代聚焦同步以下两个数据库：
- **`backtrader_web`** — 主业务库（工作区、策略单元、用户、交易配置等）
- **`akshare_data`** — 时序数据仓库（K线、行情等大量市场数据）

## 4. 技术方案

### 4.1 整体架构

```
┌──────────────────┐     HTTP API      ┌──────────────────────┐      SSH       ┌─────────────┐
│  前端             │ ───────────────→ │  后端 FastAPI         │ ──────────────→│  远程服务器   │
│  DataSyncPage.vue │                  │  sync_api.py          │               │  docker exec  │
│  /data/sync       │                  │  sync_service.py      │               │  mysqldump    │
└──────────────────┘                   └──────────────────────┘               └─────────────┘
                                              ↕
                                        本地 MySQL
                                        mysqldump / mysql
```

### 4.2 方案选型

| 方案 | 描述 | 选择 | 理由 |
|------|------|------|------|
| A: 后端 API + 前端 Tab | 后端封装同步逻辑，前端触发和展示 | ✅ | 符合项目架构，用户体验好 |
| B: 纯 CLI 脚本 | 命令行工具 | ❌ | 用户要求在 Web UI 中操作 |

底层同步引擎仍基于 `mysqldump` + `SSH` + `docker exec`，但封装为后端 API，前端通过 HTTP 调用。

### 4.3 同步流程

#### 上传（本地 → 远程）

```
1. 前端发起 POST /api/v1/data/sync/upload
2. 后端执行:
   a. 本地 mysqldump 导出 → gzip 压缩 → 临时文件
   b. scp 上传到远程 /tmp/
   c. SSH + docker exec: 在容器内创建库 + gunzip | mysql 导入
   d. 清理临时文件
3. 返回同步结果（耗时、行数等）
```

#### 下载（远程 → 本地）

```
1. 前端发起 POST /api/v1/data/sync/download
2. 后端执行:
   a. SSH + docker exec: 在容器内 mysqldump → gzip → 临时文件
   b. scp 下载到本地 /tmp/
   c. 本地 mysql 创建库 + gunzip | mysql 导入
   d. 清理临时文件
3. 返回同步结果
```

### 4.4 后端 API 设计

| 方法 | 路径 | 说明 | 优先级 |
|------|------|------|--------|
| GET | `/data/sync/config` | 获取同步配置 | P0 |
| PUT | `/data/sync/config` | 保存同步配置 | P0 |
| POST | `/data/sync/test-connection` | 测试远程连接 | P0 |
| GET | `/data/sync/databases` | 获取本地和远程数据库列表与大小 | P0 |
| POST | `/data/sync/upload` | 上传指定数据库到远程 | P0 |
| POST | `/data/sync/download` | 从远程下载数据库到本地 | P0 |
| GET | `/data/sync/status/{task_id}` | 查询同步任务进度 | P0 |
| GET | `/data/sync/history` | 获取同步历史记录 | P1 |

#### 关键 Schema

```python
# 同步配置
class SyncConfig(BaseModel):
    remote_host: str              # 远程服务器 IP/域名
    remote_user: str = "root"     # SSH 用户
    remote_ssh_key: str = "~/.ssh/id_rsa"  # SSH 密钥路径
    remote_container: str = "backtrader_mysql"  # Docker 容器名
    remote_install_dir: str = "/opt/backtrader_web"  # 部署目录
    sync_databases: list[str] = ["backtrader_web", "akshare_data"]

# 同步请求
class SyncRequest(BaseModel):
    databases: list[str]          # 要同步的数据库列表
    compress: bool = True         # 是否 gzip 压缩
    confirm: bool = False         # 是否已确认覆盖

# 同步任务状态
class SyncTaskStatus(BaseModel):
    task_id: str
    direction: str                # "upload" | "download"
    database: str
    stage: str                    # "dumping" | "transferring" | "importing" | "done" | "failed"
    progress_pct: int             # 0-100
    message: str
    started_at: str
    finished_at: str | None = None
    duration_seconds: float | None = None

# 数据库信息
class DatabaseInfo(BaseModel):
    name: str
    size_bytes: int
    size_display: str             # "42 MB", "1.2 GB"
    table_count: int
    exists: bool
```

### 4.5 同步配置持久化

同步配置保存在后端配置文件中（非数据库），路径为 `src/backend/data/sync_config.json`。

```json
{
  "remote_host": "123.45.67.89",
  "remote_user": "root",
  "remote_ssh_key": "~/.ssh/id_rsa",
  "remote_container": "backtrader_mysql",
  "remote_install_dir": "/opt/backtrader_web",
  "sync_databases": ["backtrader_web", "akshare_data"]
}
```

> 不存储任何密码。远程 MySQL 密码通过 SSH 实时从服务器 `.env` 读取；本地 MySQL 密码通过环境变量 `LOCAL_MYSQL_PASSWORD` 或后端 `.env` 中的 `SYNC_LOCAL_MYSQL_PASSWORD` 提供。

### 4.6 密码获取链路

| 角色 | 密码来源 | 说明 |
|------|----------|------|
| 本地 MySQL root | 环境变量 `SYNC_LOCAL_MYSQL_PASSWORD` 或后端 `.env` | 仅在本地运行时使用 |
| 远程 MySQL root | SSH 到服务器后读取 `.env` 中的 `MYSQL_ROOT_PASSWORD` | 自动读取，无需手动配置 |

### 6.1 新增文件

|------|------|

### 6.2 注册路由

```python

```

### 6.3 SyncService 核心方法

```python
class SyncService:
    async def test_connection(self, config: SyncConfig) -> dict
    async def get_local_databases(self) -> list[DatabaseInfo]
    async def get_remote_databases(self, config: SyncConfig) -> list[DatabaseInfo]
    async def upload_database(self, config: SyncConfig, database: str) -> SyncTaskStatus
    async def download_database(self, config: SyncConfig, database: str) -> SyncTaskStatus
```

底层通过 `asyncio.create_subprocess_exec` 调用:
- `mysqldump` — 本地/远程导出
- `gzip` / `gunzip` — 压缩/解压
- `scp` — 文件传输
- `ssh` + `docker exec` — 远程操作

### 6.4 异步执行

由于大数据库同步可能耗时数分钟，采用**后台任务**模式：
1. `POST /data/sync/upload` 立即返回 `task_id`
2. 前端轮询 `GET /data/sync/status/{task_id}` 获取进度
3. 后端使用 `asyncio.create_task()` 在后台执行同步

## 7. 安全与风险

### 7.1 安全措施

| 措施 | 说明 |
|------|------|
| 密码不入文档 | 本文档不包含任何明文密码 |
| 密码不入代码库 | 同步配置文件中不存储密码，通过环境变量或实时 SSH 读取 |
| 二次确认 | 覆盖已有库前必须前端弹窗确认 |
| SSH 密钥认证 | 优先使用 SSH 密钥，避免密码认证 |
| 管理员限制 | 数据同步 Tab 建议仅管理员可见 |
| 临时文件清理 | 同步完成后自动清理 dump 文件 |

### 7.2 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 上传覆盖远程生产数据 | 数据丢失 | 二次确认弹窗 + 显示目标库大小对比 |
| 大库传输超时 | 同步失败 | gzip 压缩 + 后台异步执行 + 超时配置 |
| SSH 连接中断 | 传输不完整 | 保留中间文件 + 前端展示失败原因 |
| Docker 容器未运行 | 无法操作远程 MySQL | 测试连接时检查容器状态并提示 |
| 字符集不匹配 | 导入乱码 | mysqldump 显式指定 `--default-character-set=utf8mb4` |
| 本地 MySQL 未运行 | 无法操作本地库 | 启动前检查本地 MySQL 连接 |

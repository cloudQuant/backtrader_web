# 迭代125 - 全面质量审查与优化迭代计划

## 1. 背景

本轮迭代基于对 `backtrader_web` 项目的**全面深度审查**（代码质量、安全、架构、测试、前端、基础设施），结合迭代 123/124 的残留问题，形成一份综合优化计划。

**项目概貌**：

- 后端：FastAPI + SQLAlchemy 2.x + Pydantic 2.x，101 个 Python 模块，约 26,800 行业务代码
- 前端：Vue 3 + TypeScript + Element Plus + ECharts + Monaco Editor，95 个 Vue/TS 文件
- 测试：97 个后端测试文件（约 29,800 行），55 个前端测试文件，Playwright E2E
- 策略库：118+ 内置策略模板
- CI：GitHub Actions，Ruff + pytest + Vitest + Playwright + Bandit + Safety

---

## 2. 调研方法

本计划基于以下**十个维度**的并行深度分析：

1. **项目结构与架构分析** — 目录布局、技术栈、模块组织
2. **后端代码质量分析** — 文件大小、函数长度、错误处理、耦合度、异步模式
3. **安全审查** — 凭证泄露、路径穿越、注入风险、认证/授权、CORS/CSP
4. **测试质量分析** — 断言强度、异常吞噬、Mock 松散度、测试隔离性
5. **API 层深度分析** — 8 个 API 文件逐行审查，重复代码、WebSocket、路径验证
6. **服务层深度分析** — 12 个服务文件逐行审查，职责违规、线程安全、事务语义
7. **基础设施层分析** — 数据库、缓存、中间件、配置、安全头、验证
8. **前端深度分析** — 30 个 Vue/TS 文件，XSS、内存泄漏、错误处理、类型安全
9. **新解耦模块分析** — 迭代 123-B 提取的 10 个模块，接口契约、签名匹配、测试覆盖
10. **测试文件逐项评分** — 13 个测试文件逐一评级，共 642+ 个测试用例

同时，本计划在落稿前对若干高风险断言做了**当前仓库状态校对**，避免把已收口事项继续写成待办：

- **[实盘主链路]** 当前主路由仍为 `app.api.live_trading_api` + `LiveTradingManager`，但旧版 `app.api.live_trading` 仍注册在 `/live-trading-crypto`，说明“遗留链路识别与处置”仍是本轮真实主题。
- **[辅助函数契约]** `load_strategy_env()` 的直接签名问题已被主链路包装隐藏，但这类“靠调用方记忆适配”的模式仍会放大后续重构风险，因此需要统一收口。
- **[提取模块测试]** 123-B 提取模块已经摆脱“零覆盖”状态，但目前覆盖主要集中在集中式隔离 happy-path；本轮应提升为模块级契约验证，而不是重复造一套形式化测试。
- **[文档漂移]** `docs/INDEX.md` 与 `docs/iterations/README.md` 尚未出现迭代125入口，且项目概述中的“全部完成”表述仍未体现能力边界，因此文档基线同步仍是需要立即收口的事项。

## 4. 迭代125目标

本迭代聚焦 **八个目标**，按紧急程度排列：

1. **紧急 Bug 与契约脆弱点修复** — 修复回调签名脆弱点、日志目录重复实现、异步/线程混用问题。
2. **安全加固** — 清理泄露凭证、加固认证、修复路径穿越/XSS。
3. **测试可信度提升** — 修复松散断言、补强提取模块契约测试与 E2E 主路径覆盖。
4. **代码质量治理** — 拆分超大文件、提取重复代码、收敛异常处理。
5. **架构边界收口** — 清理实盘遗留链路、统一任务执行模型。
6. **基础设施加固** — 缓存线程安全、数据库迁移、连接池。
7. **CI 门禁加固** — 安全扫描从 advisory 升级为 blocker。
8. **文档基线同步** — 更新快照、索引、能力矩阵。

## 7. 风险与控制

| 风险 | 控制措施 |
|------|----------|
| 凭证轮转影响正在运行的模拟交易 | 先通知后操作，确认无活跃连接后再轮转 |

| `threading.Lock → asyncio.Lock` 改动影响优化并发 | 在 staging 环境运行完整优化任务验证 |
| 大文件拆分引发 import 路径变化 | 保留旧模块的 re-export 兼容层（限期 1 个迭代后删除） |
| 遗留链路清理影响历史测试 | 先建映射表评估影响范围再操作 |
| Alembic 集成影响现有数据库 | 从现有表结构生成 initial migration，确保幂等 |

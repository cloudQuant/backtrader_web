### WP1：性能基准测试（P1）

#### 目标
建立关键API的性能基线，支持回归检测。

#### 关键API清单

| API | 重要性 | 预期基线 |
|-----|--------|----------|
| `POST /api/v1/auth/login` | 高 | < 100ms |
| `POST /api/v1/auth/register` | 高 | < 150ms |
| `GET /api/v1/strategies` | 高 | < 50ms |
| `POST /api/v1/backtests/run` | 高 | < 200ms (不含回测执行) |
| `GET /api/v1/backtests/{id}` | 高 | < 50ms |
| `GET /api/v1/health` | 中 | < 10ms |

#### 执行步骤

1. **创建性能测试文件**：
   ```python
   # tests/test_performance_baseline.py
   import pytest
   import time
   
   @pytest.mark.performance
   async def test_login_performance(client):
       start = time.time()
       # ... login request
       duration = time.time() - start
       assert duration < 0.1, f"Login took {duration}s, expected < 0.1s"
   ```

2. **添加性能测试配置**：
   ```ini
   # pytest.ini
   [pytest]
   markers =
       performance: Performance benchmark tests
   ```

3. **CI集成（可选）**：
   - 在CI中运行性能测试
   - 记录性能指标趋势

### WP2：安全审计（P1）

#### 目标
建立自动化依赖安全扫描流程。

#### 执行步骤

1. **后端依赖扫描**：
   ```bash
   # 使用 pip-audit 扫描Python依赖
   pip install pip-audit
   pip-audit --desc
   ```

2. **前端依赖扫描**：
   ```bash
   # 使用 npm audit 扫描Node依赖
   npm audit
   npm audit fix
   ```

3. **添加安全扫描脚本**：
   ```bash
   # scripts/security_scan.sh
   #!/bin/bash
   echo "=== Python Security Audit ==="
   pip-audit --desc
   echo "=== Node Security Audit ==="
   npm audit
   ```

4. **更新pre-commit配置**：
   ```yaml
   # .pre-commit-config.yaml
   - repo: https://github.com/Lucas-C/pre-commit-hooks-safety
     rev: v1.3.3
     hooks:
       - id: python-safety-dependencies-check
   ```

### WP4：监控告警（P2）

#### 目标
定义关键指标告警阈值。

#### 关键指标与阈值

| 指标 | 告警阈值 | 严重级别 |
|------|----------|----------|
| API响应时间 | > 5s | Warning |
| API响应时间 | > 10s | Critical |
| 错误率 | > 5% | Warning |
| 错误率 | > 10% | Critical |
| 数据库连接数 | > 80% | Warning |
| 内存使用 | > 85% | Warning |
| CPU使用 | > 90% | Critical |

#### 执行步骤

1. **创建告警配置文件**：
   ```yaml
   # config/alerting.yaml
   alerts:
     - name: high_latency
       metric: http_request_duration_seconds
       threshold: 5.0
       severity: warning
     - name: high_error_rate
       metric: http_requests_failed_total
       threshold: 0.05
       severity: warning
   ```

2. **添加告警规则文档**：
   - 在 `docs/` 中添加告警配置说明
   - 记录各指标的阈值定义

3. **集成Prometheus告警（可选）**：
   - 创建 `prometheus/alerts.yml`

## Phase 1: 修复阻塞性问题 (P0)

### 1.1 修复后端测试隔离问题

**问题**: `test_integration_backtest_flow.py` 在全量运行时 FAILED，单独运行时 PASSED，说明存在测试间状态泄漏。

**根因分析方向**:
- 检查是否有全局状态（如数据库、单例服务）在测试间共享
- 检查是否缺少 fixture 的 teardown/cleanup
- 检查与前面测试的执行顺序依赖

**修复方案**:
1. 使用 `pytest --lf -v` 定位确切的前序污染测试
2. 确保集成测试使用独立的数据库 session
3. 添加 fixture 清理逻辑

**文件**: `src/backend/tests/test_integration_backtest_flow.py`, `src/backend/tests/conftest.py`

### 2.2 统一日志框架

**问题**: 项目声称使用 loguru，但 ~35 个文件使用 `import logging` + `logging.getLogger(__name__)`，仅 `main.py` 和少数文件使用 `setup_logger`/loguru。

**修复方案**:
1. 确定统一方案：在 `utils/logger.py` 中通过 loguru 的 `intercept` 机制将 stdlib logging 转发到 loguru
2. 保持各文件使用 `logging.getLogger(__name__)` 不变（避免大规模改动），让 loguru 统一处理输出格式
3. 验证日志在 DEBUG/PRODUCTION 模式下输出一致

**文件**: `src/backend/app/utils/logger.py`

## Phase 4: 测试基础设施改善 (P2)

### 4.1 注册自定义 pytest mark

**问题**: `pytest.mark.performance` 未在 `pytest.ini` 中注册，产生 4 条警告。

**修复**: 在 `pytest.ini` 中添加:
```ini
markers =
    performance: marks tests as performance benchmarks
```

### 4.2 修复前端 Sass 弃用警告

**问题**: `Deprecation [legacy-js-api]: The legacy JS API is deprecated`

**修复**: 在 `vite.config.ts` 中配置 Sass 使用新 API:
```typescript
css: {
  preprocessorOptions: {
    scss: { api: 'modern-compiler' }
  }
}
```

### WP5：前端错误处理链路统一（P1）

#### 目标
统一前端错误提示行为，避免重复弹窗或静默。

#### 当前问题

根据迭代129分析：
- `src/frontend/src/api/index.ts` 响应拦截器统一弹错
- `BacktestPage.vue` 中 `showRequestMessage()` 对 Axios 错误直接 `return`
- 导致部分 warning 场景被吞掉

#### 执行步骤

1. 梳理错误分类：
   - 网络/Axios 错误 → 全局处理
   - 业务逻辑错误 → 局部处理
   - 回退/超时警告 → 局部处理

2. 明确职责边界：
   - 全局拦截器：处理 HTTP 级别错误（4xx, 5xx）
   - 局部处理：处理业务级别提示（warning, info）

3. 修改 `showRequestMessage()` 逻辑，避免吞掉 warning

#### 验收标准

- [x] 错误分类清晰，职责边界明确
- [x] 不存在重复弹窗
- [x] warning 场景能正常显示

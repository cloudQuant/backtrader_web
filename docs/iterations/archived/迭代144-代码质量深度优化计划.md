### WP2：BacktestService代码重构（P2）

#### 目标
消除`BacktestService`中的重复代码，提升可维护性。

#### 问题分析
`_build_backtest_result`和`get_result`存在重复的结果构建逻辑：

```python
# 当前：两处重复的结果构建代码
# 位置1: _build_backtest_result (静态方法)
# 位置2: get_result (实例方法)
```

#### 重构方案
```python
# 统一使用静态方法构建结果
@staticmethod
def _build_backtest_result(
    task: BacktestTask, 
    result_model: BacktestResultModel | None
) -> BacktestResult:
    ...

async def get_result(self, task_id: str, user_id: str | None = None) -> BacktestResult | None:
    task = await self.task_repo.get_by_id(task_id)
    if not task:
        return None
    if user_id and task.user_id != user_id:
        return None
    
    # 使用统一的结果构建方法
    result_model = await self._get_result_model(task_id)
    return self._build_backtest_result(task, result_model)
```

#### 验收标准
- 代码重复行数减少50%+
- 所有现有测试通过
- 无功能回归

| StrategyService | 混合模式 | 需统一 |
| LiveTradingService | 返回None/False | ✅ 已规范 |

#### 标准化方案
```python
# 服务层标准模式
class ServiceResult[T]:
    """统一的服务返回类型"""
    success: bool
    data: T | None
    error: str | None
    
# 或保持现有模式但添加文档
async def get_strategy(...) -> Strategy | None:
    """获取策略。
    
    Returns:
        Strategy: 找到时返回
        None: 未找到或无权限
    """
```

### WP5：缓存策略优化（P3）

#### 目标
优化高频查询的缓存策略，提升响应性能。

#### 当前缓存使用
```python
# 已缓存
- backtest:result:{task_id}  # TTL: 3600s

# 可优化缓存
- 策略模板列表
- 用户策略列表
- 回测任务列表
```

#### 优化方案
```python
# 策略模板缓存（变化频率低）
CACHE_KEY_TEMPLATES = "strategy:templates"
CACHE_TTL_TEMPLATES = 86400  # 24小时

# 用户策略列表缓存（按用户）
CACHE_KEY_USER_STRATEGIES = "user:{user_id}:strategies"
CACHE_TTL_USER_STRATEGIES = 300  # 5分钟
```

#### 验收标准
- 高频查询响应时间降低30%+
- 缓存命中率可监控

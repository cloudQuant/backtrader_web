## Phase 3: 异常处理与日志改进 (P1)

### 任务 3.1: 修复异常上下文丢失

**问题**: 多处 `except Exception` 捕获后未保留完整栈信息：
- `main.py:218` — 健康检查异常无上下文
- `websocket_manager.py:35-36` — 返回 False 不记录栈
- `backtest_enhanced.py:101` — debug 级别记录关键错误

**修复方案**:
- [ ] 所有 `except Exception as e` 改为 `logger.exception()` 或 `logger.error(..., exc_info=True)`
- [ ] 关键路径（回测执行、实盘交易、优化）异常级别提升到 `error`
- [ ] WebSocket 异常发送标准错误帧给客户端而非静默失败

### 任务 3.2: 统一 WebSocket 错误消息格式

**文件**: `src/backend/app/websocket_manager.py`

**修复方案**:
- [ ] 定义标准 WebSocket 错误消息格式：`{ "type": "error", "code": "...", "message": "..." }`

## 执行顺序与依赖关系

```
Phase 1 (测试修复) ──→ Phase 2 (API 一致性) ──→ Phase 3 (异常处理)
     ↓                                              ↓
Phase 5 (CI/CD) ←──────────────────────────── Phase 4 (状态管理)
     ↓
Phase 6 (E2E)
```

**关键原则**:
- Phase 1 的测试修复最优先，确保测试套件可信赖
- Phase 2-4 可根据实际情况并行推进
- Phase 5 CI 改进应在测试修复后进行，避免 CI 误报
- Phase 6 E2E 优先级最低，但对发布信心影响大

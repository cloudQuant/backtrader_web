# 迭代127 - 策略交易工作区（实盘交易工作区）

> **文档状态**: 优化中  
> **创建日期**: 2026-04-13  
> **迭代目标**: 将现有"模拟交易"和"实盘交易"合并为统一的"策略交易"工作区，复用策略研究架构

## 1. 迭代概述

本迭代的核心目标是将现有独立的"模拟交易"和"实盘交易"功能整合为统一的**策略交易**功能模块。新功能采用与"策略研究"一致的工作区架构，支持工作区管理、策略单元管理、实时交易状态监控、头寸/盈亏追踪等能力。

### 核心变更

1. **新增"策略交易"侧边栏入口** — 首期先与旧入口并存，验证稳定后再替换现有"模拟交易"和"实盘交易"入口
2. **复用工作区架构** — 基于迭代124的 Workspace + StrategyUnit 模型扩展交易能力
3. **统一交易管理** — 在同一个工作区内管理模拟交易和实盘交易的策略单元
4. **实时状态监控** — 多仓/空仓、盈亏、最新价格、杠杆等实时交易指标展示
5. **策略研究互通** — 从策略研究导出的策略单元可直接导入策略交易

### 后续阶段删除项

- 侧边栏"模拟交易"入口及 `/simulate` 路由
- 侧边栏"实盘交易"入口及 `/live-trading` 路由
- 对应的旧页面组件（SimulatePage.vue、LiveTradingPage.vue 等）

> 上述删除项不作为首批上线门槛，待新“策略交易”入口通过业务验收后再执行。

## 4. 关键决策记录

| 决策 | 选项 | 结论 | 理由 |
|------|------|------|------|
| 工作区模型复用 | A: 扩展现有Workspace表 / B: 新建独立表 | **A** | 最大化代码复用，便于策略研究→策略交易导入 |
| 交易运行时状态 | A: 入库持久化 / B: 内存+Redis | **B优先，A降级** | 交易运行时数据变化频繁，持久化开销大；内存模式足够单机部署 |
| 前端组件复用 | A: 完全复用策略研究组件 / B: 独立开发 | **A为主** | 需求明确要求样式一致，工具栏结构高度相似 |
| 自动交易调度 | A: 嵌入工作区 / B: 独立服务 | **A（P1接入）** | 现有 LiveTradingPage 已有自动交易调度，但首批上线优先保证手动交易闭环 |

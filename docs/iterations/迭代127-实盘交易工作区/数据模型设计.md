# 迭代127 - 策略交易工作区 - 数据模型设计

> **文档状态**: 初版  
> **最后更新**: 2026-04-13  
> **核心策略**: 扩展现有 Workspace/StrategyUnit 模型，交易运行时状态采用内存管理

## 1. 模型扩展策略

本迭代 **不新建独立的交易工作区表**，而是在现有迭代124的 Workspace 和 StrategyUnit 模型上进行扩展：

- `Workspace` 新增 `workspace_type` 字段区分 `research`（策略研究）和 `trading`（策略交易）
- `StrategyUnit` 新增交易模式、网关配置、锁定状态等持久化字段
- 交易运行时状态（头寸/盈亏/价格）通过内存 `TradingRuntimeManager` 管理，不频繁入库
- 新增 `TradingAutoConfig` 表存储自动交易调度配置

## 3. 新增表

### 3.1 trading_auto_configs（自动交易调度配置表）

```sql
CREATE TABLE trading_auto_configs (
    id              VARCHAR(36)  PRIMARY KEY,
    workspace_id    VARCHAR(36)  NOT NULL UNIQUE,
    user_id         VARCHAR(100) NOT NULL,
    
    enabled         BOOLEAN      DEFAULT FALSE,
    buffer_minutes  INT          DEFAULT 15,
    scope           VARCHAR(20)  DEFAULT 'all',
    -- 'all'        = 所有单元
    -- 'live'       = 仅实盘
    -- 'simulate'   = 仅模拟
    
    sessions        JSON         NOT NULL,
    -- [{"name": "日盘", "open": "09:00", "close": "15:00"}, ...]
    
    created_at      DATETIME     NOT NULL,
    updated_at      DATETIME     NOT NULL,

    FOREIGN KEY (workspace_id) REFERENCES workspaces(id) ON DELETE CASCADE,
    INDEX idx_auto_config_user (user_id)
);
```

### 3.2 trading_daily_summaries（交易日汇总表，P1）

```sql
CREATE TABLE trading_daily_summaries (
    id              VARCHAR(36)  PRIMARY KEY,
    unit_id         VARCHAR(36)  NOT NULL,
    workspace_id    VARCHAR(36)  NOT NULL,
    trading_date    DATE         NOT NULL,
    
    -- 当日数据
    open_pnl        FLOAT        DEFAULT 0,
    close_pnl       FLOAT        DEFAULT 0,
    daily_pnl       FLOAT        DEFAULT 0,
    trade_count     INT          DEFAULT 0,
    long_trades     INT          DEFAULT 0,
    short_trades    INT          DEFAULT 0,
    max_position    INT          DEFAULT 0,
    
    -- 累计数据
    cumulative_pnl  FLOAT        DEFAULT 0,
    max_drawdown    FLOAT        DEFAULT 0,
    
    created_at      DATETIME     NOT NULL,

    FOREIGN KEY (unit_id) REFERENCES strategy_units(id) ON DELETE CASCADE,
    FOREIGN KEY (workspace_id) REFERENCES workspaces(id) ON DELETE CASCADE,
    UNIQUE INDEX idx_daily_unit_date (unit_id, trading_date),
    INDEX idx_daily_workspace (workspace_id, trading_date)
);
```

## 4. 内存运行时状态模型

交易运行时高频变化数据不入库，由 `TradingRuntimeManager` 在内存中管理。

### 4.1 TradingUnitRuntimeState

```python
@dataclass
class TradingUnitRuntimeState:
    """单个策略单元的交易运行时状态"""
    unit_id: str
    workspace_id: str
    
    # 运行状态
    is_running: bool = False
    trading_mode: str = "simulate"      # simulate / live
    instance_id: str | None = None      # 关联的交易实例ID
    started_at: datetime | None = None
    error: str | None = None
    
    # 头寸
    long_position: int = 0
    short_position: int = 0
    
    # 盈亏
    daily_pnl: float = 0.0
    position_pnl: float = 0.0
    total_pnl: float = 0.0
    
    # 价格
    latest_price: float = 0.0
    price_change_pct: float = 0.0
    
    # 市值与风控
    long_market_value: float = 0.0
    short_market_value: float = 0.0
    leverage: float = 0.0
    max_drawdown_rate: float = 0.0
    
    # 统计
    bar_count: int = 0
    trading_days: int = 0
    
    # 锁定
    lock_trading: bool = False
    lock_running: bool = False
    
    # 更新时间
    last_update: datetime | None = None
```

### 4.2 TradingRuntimeManager

```python
class TradingRuntimeManager:
    """交易运行时状态管理器（单例）"""
    
    _states: dict[str, TradingUnitRuntimeState]  # key = unit_id
    _lock: asyncio.Lock
    
    async def get_unit_state(self, unit_id: str) -> TradingUnitRuntimeState | None
    async def get_workspace_states(self, workspace_id: str) -> list[TradingUnitRuntimeState]
    async def update_state(self, unit_id: str, **kwargs) -> None
    async def remove_state(self, unit_id: str) -> None
    
    # 从交易引擎回调更新
    async def on_order_fill(self, unit_id: str, fill_data: dict) -> None
    async def on_price_update(self, unit_id: str, price: float, change_pct: float) -> None
    async def on_position_change(self, unit_id: str, long: int, short: int) -> None
    
    # 定期持久化快照到数据库
    async def persist_snapshots(self) -> None  # 每5分钟调用一次
    
    # 启动时从数据库恢复
    async def restore_from_db(self, workspace_id: str) -> None
```

## 8. 数据迁移策略

### 8.1 Alembic 迁移脚本

```python
# 迁移步骤：
# 1. ALTER workspaces 新增 workspace_type, trading_config 字段
# 2. ALTER strategy_units 新增 trading_mode, gateway_config, lock_*, trading_instance_id, trading_snapshot 字段

# 4. CREATE TABLE trading_daily_summaries (P1)
# 5. UPDATE workspaces SET workspace_type = 'research' WHERE workspace_type IS NULL
```

### 8.2 旧数据迁移

现有模拟交易和实盘交易的实例数据（`live_trading_instances.json`）可选择性迁移为策略交易工作区中的策略单元。此步骤为可选，因为旧系统以实例方式管理（内存态），新系统以工作区+数据库方式管理。

## 9. 与现有数据的关系

```
Workspace (workspace_type='trading')
  │
  ├── StrategyUnit (trading_mode='simulate')
  │     └── trading_instance_id → PaperTradingService 内存实例
  │
  ├── StrategyUnit (trading_mode='live')
  │     ├── gateway_config → GatewayPreset 引用
  │     └── trading_instance_id → LiveTradingService 内存实例
  │
  └── TradingAutoConfig → 自动交易调度
        └── sessions → 交易时段定义

TradingRuntimeManager (内存)
  └── TradingUnitRuntimeState
        ├── 头寸/盈亏/价格 → 实时更新
        └── 定期 → trading_snapshot (持久化)
```

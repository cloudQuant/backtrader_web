### 关键Bug

**P0: 优化使用策略模板目录而非单元运行目录**

`submit_unit_optimization` 中：
```python
strategy_id = unit.strategy_id or ""
submit_optimization(strategy_id=strategy_id, ...)
```
这会通过 `get_strategy_dir(strategy_id)` 获取策略模板目录，而非单元的运行目录。
导致优化试验不使用单元配置的 symbol/timeframe/data_config，而是用策略模板的默认配置。

**修复方案**：在 `submit_unit_optimization` 中使用 `workspace_unit_runtime.unit_dir(workspace_id, unit_id)` 获取单元的实际运行目录，并确保 `sync_unit_runtime` 已执行。

## 2. 实现计划

### Phase 1: 修复核心优化流程（P0）

1. 修改 `workspace_service.submit_unit_optimization`：
   - 先调用 `sync_unit_runtime` 确保单元目录就绪
   - 将 unit_dir 传给 `submit_optimization` 而非 strategy_id
   - 在 `submit_optimization` 中支持直接传入 `strategy_dir` 参数

2. 修改 `param_optimization_service.submit_optimization`：
   - 新增可选参数 `strategy_dir: str | None`
   - 如果传入 strategy_dir，跳过 `get_strategy_dir` 查找

### Phase 2: 增强试验指标（P1）

扩展 `parse_trial_logs` 返回更多指标：
- `initial_cash`, `net_value`, `net_profit`
- `profit_factor`, `avg_profit`, `avg_profit_rate`
- `total_win_amount`, `total_loss_amount`, `profit_loss_ratio`
- `win_rate` (已有), `odds`
- `daily_avg_return`, `daily_max_loss`, `daily_max_profit`
- `weekly_avg_return`, `weekly_max_loss`, `weekly_max_profit`
- `monthly_avg_return`, `monthly_max_loss`, `monthly_max_profit`
- `trading_days`, `trading_cost`

### Phase 3: 批量优化（P2）

1. **批量参数设置**：新建 `BatchOptimizationConfigDialog.vue`
   - 选中多个单元后，统一设置优化参数（param_layers, objective 等）
   - 保存到每个选中单元的 `optimization_config`

2. **批量提交优化**：在 WorkspaceUnitsTab 添加"批量提交优化"按钮
   - 遍历选中单元，逐个调用 `submitOptimization`
   - 在 WorkspaceOptimizationTab 中支持按单元切换查看结果

### Phase 4: 优化结果跳转回测详情（P3）

1. 在优化结果表格行添加"查看详情"按钮
2. 点击后，使用该行的 params 对单元执行一次回测
3. 回测完成后跳转到 `/backtest/result/{task_id}` 页面

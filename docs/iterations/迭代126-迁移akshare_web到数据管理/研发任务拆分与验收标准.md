## Epic 1：后端模型与 Schema 迁移

### Story 1.0 建立双数据库 engine 基建（新增，前置）

> 这是 Epic 1 的前置 Story，必须先于 1.1 完成。

建议产出：

- `src/backend/app/db/database.py`
- `src/backend/app/db/akshare_data_database.py`
- 更新 `DATABASE_URL` 默认与说明到 `app/config.py`
- 新增迁移源配置（如 `LEGACY_SQLITE_DATABASE_URL`、`LEGACY_AKSHARE_DATABASE_URL`）
- 新增 akshare 运行配置（如 `AKSHARE_DATA_DATABASE_URL`、`AKSHARE_SCHEDULER_TIMEZONE`、`AKSHARE_SCRIPT_ROOT`、`AKSHARE_INTERFACE_BOOTSTRAP_MODE`）
- 更新 `.env` 与 `.env.example`
- 安装依赖 `aiomysql`（或等价异步 MySQL 驱动）
- 独立的 `get_akshare_data_session` 依赖注入函数
- 建库脚本 / 初始化脚本（若 `backtrader_web` 不存在则创建）

最低要求：

- 运行时主库 session 与 `akshare_data` session 完全隔离
- 可通过依赖注入方式在 API 层获取 `akshare_data` session
- `akshare_data` 库无连接时，不影响主库基本启动（失败降级友好）
- 启动日志明确区分"主库连接成功"与"akshare_data 库连接成功"

DoD：

- 开发环境可同时运行 `backtrader_web` MySQL 与 `akshare_data` MySQL
- 主业务模型与 akshare 管理模型可在 `backtrader_web` MySQL 中创建与查询
- `akshare_data` 继续物理独立，不与主库混表
- SQLite 被明确降级为迁移源，而不是运行依赖

### Story 1.1 建立 akshare 数据模型

建议产出：

- `src/backend/app/models/akshare_mgmt.py`

最低要求：

- 包含 7 个核心实体（见总体方案 6.1）
- 落在 `backtrader_web` 主库 metadata 中
- 时间字段统一使用 `DateTime`，存 UTC
- 用户字段（`created_by` / `updated_by` / `user_id` / `operator_id`）与主库 `users` 体系兼容
- **必须遵循总体方案 6.4 前向兼容设计原则**：
  - `ak_data_tables` 包含 `symbol_raw`、`symbol_normalized`、`market`、`asset_type`
  - `ak_data_tables` 包含 `metadata` JSON 字段
  - `ak_data_interfaces` 包含 `extra_config` JSON 字段
  - 任何涉及 OHLCV 的字段使用行业通用命名

DoD：

- 模型可导入
- 模型可参与 `backtrader_web` 主库建表
- 与用户关联字段已适配当前用户体系
- 前向兼容字段齐全，后续迭代可通过"新增表"而非"改表结构"来对接回测

### Story 1.2 建立 Schema

建议产出：

DoD：

### Story 1.3 编写 Alembic 与初始化脚本

最低要求：

- 在 `backtrader_web` MySQL 中建立主业务 schema 与 7 张 akshare 管理表

DoD：

- `backtrader_web` MySQL `upgrade` 可执行

### Story 1.4 编写数据迁移脚本（新增）

- 无法迁移记录有失败清单与人工处理建议
- 应用切到 MySQL 后，不再依赖 SQLite 文件正常运行

## 7.5 端到端场景验收（产品视角）

本迭代至少需要跑通以下**6 个端到端场景**，才能算产品层面交付完成：

### 场景 1：管理员创建数据接口 → 创建任务 → 执行 → 查看记录

```text
登录（管理员）
-> /data/scripts 创建一个数据接口（例如 A 股日线采集脚本）
-> /data/tasks 基于该接口创建一个定时任务
-> 点击"手动执行"
-> /data/executions 能查到刚才的执行记录，状态为成功或失败
-> 若成功，/data/tables 能看到新生成的数据表，可预览前若干行数据
```

### 场景 2：任务启停

```text
/data/tasks 选中一个启用中的任务
-> 点击停止
-> 调度器中该任务不再触发
-> 点击启动
-> 下次调度时间前任务被正确注册
-> 数据库中 is_active 字段与调度器状态一致
```

### 场景 3：非管理员权限限制

```text
登录（普通用户）
-> 侧边栏仍可看到"数据管理"
-> 进入 /data 后，顶部 Tab 中"接口管理"不可见或被禁用
-> 数据接口、任务、数据表等页面中，创建 / 编辑 / 删除按钮不可见
-> 直接请求后端写操作 API 返回 403
```
